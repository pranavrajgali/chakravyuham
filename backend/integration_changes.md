# Chakravyuham Integration Changes

This document lists all modifications made to migrate the **Chakravyuham CAN Bus Intrusion Detection System** from a Streamlit prototype to a decoupled React + FastAPI production-grade cyber security monitor.

---

## 1. Files Modified and Created

### 📂 Created: `server.py`
*   **Location:** `C:\Users\Admin\Documents\Projects\Chakravyuham\kerintha_ids\server.py`
*   **Description:** Built a FastAPI WebSocket backend streaming CAN frame telemetry and predictions.
*   **Features Included:**
    *   Loads your trained XGBoost model (`results/suite_otids/xgboost/model.pkl`) on startup.
    *   Tracks sequential features statefully per client socket connection: Inter-Arrival Time (IAT), Jitter (IAT variation), Global Message Frequency, Shannon Entropy, and Hamming Distance.
    *   Generates background traffic matching attack modes (`DOS`, `FUZZ`, `SPOOF`, `NORMAL`) dynamically.
    *   Processes features in real-time, runs model inference, and broadcasts predicted verdicts to connected UI clients.

### 📂 Modified: `frontend/src/App.tsx`
*   **Location:** `C:\Users\Admin\Documents\Projects\Chakravyuham\kerintha_ids\frontend\src\App.tsx`
*   **Description:** Connected the React interface to the backend.
*   **Changes Made:**
    *   Added a WebSocket client listener connecting to `ws://127.0.0.1:8000/ws/can`.
    *   Muted the local browser-side simulation ticks (`setMuteSimulation(true)`) to rely entirely on predictions generated by the Python backend.

## 2. Server Run Commands

To host this decoupled system locally:

### 🐍 Run the Backend Server
Make sure you are in the virtual environment `.venv` and execute:
```bash
.venv\Scripts\python -m uvicorn server:app --host 127.0.0.1 --port 8000 --reload
```

### ⚡ Run the React Frontend
Install the client-side packages and run the Vite compiler:
```bash
cd frontend
npm install
npm run dev
```
Open **`http://localhost:3000`** in your browser.
